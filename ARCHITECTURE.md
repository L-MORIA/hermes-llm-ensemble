# Архитектура Hermes LLM Ensemble

## Общая схема

```
Пользователь: задача + режим (судья/синтезатор/мозговой штурм)
    │
    ▼
┌─────────────────────────────────────────────┐
│              ДИСПЕТЧЕР (llm-ensemble)        │
│  • Принимает задачу                          │
│  • Определяет режим арбитра                  │
│  • Рассылает задачу N агентам                │
└───────────────┬─────────────────────────────┘
    │
    ├──────┬──────┬──────┐
    ▼      ▼      ▼      ▼
┌──────┐┌──────┐┌──────┐┌──────────┐
│Agent ││Agent ││Agent ││ (резерв) │
│  A   ││  B   ││  C   ││          │
│Deep- ││MiMo  ││Big   ││North Mini│
│Seek  ││V2.5  ││Pickle││          │
└──┬───┘└──┬───┘└──┬───┘└──────────┘
   │       │       │
   ▼       ▼       ▼
  Реш.1   Реш.2   Реш.3
   │       │       │
   └───────┼───────┘
           ▼
┌─────────────────────────────────────────────┐
│             АРБИТР (Nemotron-3 Ultra)        │
│                                             │
│  ⚖️ Судья → выбирает лучшее                  │
│  🧩 Синтезатор → объединяет лучшее из каждого│
│  💡 Мозговой штурм → рождает новое           │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
        ┌─────────────────────┐
        │  ИТОГОВОЕ РЕШЕНИЕ   │
        │  + обоснование      │
        └─────────────────────┘
```

## Компоненты

### 1. Диспетчер (скил llm-ensemble)

Роль: координация. Реализован как SKILL.md — инструкция для Hermes Agent.

Отвечает за:
- Парсинг задачи и режима
- Выбор моделей для агентов
- Запуск `delegate_task(tasks=[...])` — до 3 агентов параллельно
- Формирование промпта для арбитра
- Вывод результата пользователю

### 2. Агенты (OpenCode CLI)

Каждый агент — это `opencode run '...' --model <model>`.

| Агент | Модель | Провайдер |
|-------|--------|-----------|
| A | `opencode/deepseek-v4-flash-free` | OpenCode Zen |
| B | `opencode/mimo-v2.5-free` (Xiaomi MiMo) | OpenCode Zen |
| C | `opencode/big-pickle` | OpenCode Zen |

Процесс агента:
1. Получает задачу (текст)
2. Решает её полностью самостоятельно
3. Возвращает код/текст + пояснение

### 3. Арбитр (Nemotron-3 Ultra)

Самая мощная модель (550B, 1M контекст). Запускается через `opencode run --model opencode/nemotron-3-ultra-free`.

Три режима работы:

| Режим | Промпт | Результат |
|-------|--------|-----------|
| ⚖️ Судья | «Выбери ОДНО лучшее решение» | Один вариант |
| 🧩 Синтезатор | «Объедини ЛУЧШЕЕ ИЗ КАЖДОГО» | Гибрид |
| 💡 Мозговой штурм | «Создай НОВОЕ решение» | Новая концепция |

### 4. Параллельность

```
max_concurrent_children: 3 (Hermes config)
```

- 3 агента запускаются ПАРАЛЛЕЛЬНО через `delegate_task(tasks=[...])`
- Арбитр запускается ПОСЛЕДОВАТЕЛЬНО (должен видеть все решения)
- 4-й агент не даёт прироста (лимит 3 + diminishing returns)

## Поток данных

```
Задача (текст)
  → [Диспетчер] форматирует промпты
    → [delegate_task x3] параллельные вызовы OpenCode
      → [Агенты] решения
    → [Арбитр] оценивает
  → [Диспетчер] выводит результат
```

## Технические детали

### OpenCode Zen

- Провайдер: `opencode`
- Авторизация: не требуется (работает из коробки)
- Скорость: зависит от очереди (обычно 2-10 сек на ответ)
- Модели доступны: `opencode models opencode`

### Ограничения

- CPU: i5-3337U, 2 ядра / 4 потока — не влияет на OpenCode (API)
- RAM: 8 GB — достаточно
- Hermes `max_concurrent_children: 3` — макс 3 параллельных агента
- OpenCode: таймауты при большой очереди — повторный запуск решает
